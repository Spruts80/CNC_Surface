# G-Code Studio - Генератор и Визуализатор G-кода для ЧПУ

## 📋 Описание проекта

**G-Code Studio** — это открытое Python-приложение для генерации и визуализации G-кода для фрезерных станков с ЧПУ. Программа предоставляет графический интерфейс для настройки параметров обработки, генерации управляющих программ и их визуальной проверки перед запуском на реальном оборудовании.

---

## ✨ Основные возможности

### 🔧 Генератор G-кода
- **4 типа обработки:**
  - Зигзаг по оси X
  - Зигзаг по оси Y
  - От центра змейкой (спиральная обработка)
  - Контурное фрезерование (от края к центру / от центра к краю)
- **Настройка параметров:**
  - Размеры рабочего поля (X, Y)
  - Диаметр фрезы и шаг между проходами (stepover)
  - Глубина обработки (Z start, Z end, Z step)
  - Безопасная высота Z
  - Подачи XY и Z
  - Скорость шпинделя
  - Номер инструмента
- **Автоматическая разбивка на файлы** по количеству проходов
- **Валидация параметров** с проверкой корректности настроек
- **Расчёт перекрытия фрезы** в процентах

### 📊 Визуализатор
- **2D проекции:** XY, XZ, YZ
- **Анимация обработки** с отображением:
  - Текущих координат фрезы (X, Y, Z)
  - Типа движения (G0 - холостой ход, G1 - рабочий ход)
  - Прогресса выполнения
  - Скорости анимации (1-50 кадров)
- **Цветовое кодирование:**
  - Красный — холостые перемещения (G0)
  - Синий — рабочие перемещения (G1)
  - Зелёный — обработанная область
- **Отображение фрезы** с учётом диаметра инструмента

### 📝 Работа с файлами
- **Просмотр G-кода** в отдельном окне
- **Поиск по коду** с подсветкой найденных строк
- **Сохранение и копирование** G-кода
- **Открытие папки** с сгенерированными файлами
- **Логирование операций** с возможностью сохранения

## ❌ Минусы (Ограничения и недостатки)

### 🔴 Критические ограничения
1. **Только 2D визуализация** — нет полноценного 3D просмотра траектории
2. **Нет симуляции съёма материала** — не показывает результат обработки заготовки
3. **Нет проверки коллизий** — не предупреждает о столкновениях фрезы с заготовкой или приспособлениями
4. **Нет поддержки G2/G3** — только линейные перемещения (G0, G1), нет круговой интерполяции

### 🟡 Функциональные недостатки
1. **Один инструмент** — нет поддержки автоматической смены инструментов
2. **Нет сохранения настроек** — параметры сбрасываются при перезапуске
3. **Нет импорта CAD/CAM** — только ручная настройка параметров
4. **Ограниченные типы обработки** — нет:
   - 3D обработки поверхностей
   - Сверления отверстий
   - Нарезания резьбы
   - Гравировки текста/изображений
5. **Нет оптимизации траектории** — не минимизирует холостые перемещения
6. **Нет расчёта времени обработки**

### 🟠 Технические проблемы
1. **Дублирование кода в UI** — блок параметров Z повторяется дважды
2. **Неоптимальная генерация** — в некоторых режимах создаются лишние команды
3. **Анимация только в XY** — в проекциях XZ/YZ анимация не работает корректно
4. **Нет многопоточности** — генерация больших файлов может блокировать интерфейс
5. **Отсутствие тестов** — нет автоматических тестов для проверки корректности

### 🟢 Мелкие недостатки
1. **Нет локализации** — интерфейс только на русском языке
2. **Нет документации** — отсутствует подробное руководство пользователя
3. **Нет примеров** — нет готовых пресетов для типовых задач
4. **Примитивный дизайн** — интерфейс функциональный, но не современный

## 📦 Требования для запуска

### Системные требования
- **Python 3.7+**
- **Операционная система:** Windows 10+, macOS 10.14+, Linux (Ubuntu 18.04+)
- **RAM:** минимум 512 МБ (рекомендуется 1 ГБ)
- **Диск:** ~50 МБ свободного места

### Зависимости
```bash
pip install matplotlib numpy
```

**Опционально:**
```bash
pip install pygcode  # для расширенного парсинга G-кода
```

## 🏆 Заключение

**G-Code Studio** — это функциональный инструмент для базовой генерации и визуализации G-кода, подходящий для обучения и простых задач фрезерования. Программа успешно решает поставленные задачи, но имеет ограничения для профессионального использования.

**Рекомендуется для:**
- ✅ Обучения основам ЧПУ
- ✅ Прототипирования и хобби-проектов
- ✅ Быстрой генерации простых траекторий
- ✅ Визуальной проверки G-кода

**Не рекомендуется для:**
- ❌ Сложных 3D поверхностей
- ❌ Промышленного производства
- ❌ Многоинструментальной обработки
- ❌ Задач с высокими требованиями к точности

Проект имеет хороший потенциал для развития и может стать полноценной CAM-системой при добавлении 3D визуализации, симуляции съёма материала и расширении типов обработки.
